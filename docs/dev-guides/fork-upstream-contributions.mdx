This fork tracks [the-momentum/open-wearables](https://github.com/the-momentum/open-wearables) as `origin`. The items below are **generic** improvements already in this fork that upstream would benefit from. Keep fork-only work (MCP OAuth connector, coach app, calendar-day API parsing) isolated — see the **Fork extensions** nav group.

## Sync workflow

```bash
git fetch origin
git checkout main
git merge origin/main
# resolve conflicts; run make migrate && cd backend && uv run pytest -v
git checkout feat/your-branch && git merge main
```

A weekly [GitHub Action](https://github.com/CalvinKorver/open-wearables/blob/main/.github/workflows/upstream-sync-check.yml) on this fork reports when `main` is behind upstream.

## PR 1: OAuth configuration guards

**Files:** `backend/app/services/providers/templates/base_oauth.py`

**Summary:** Fail fast with `503` when OAuth client credentials or production redirect URI are missing, instead of redirecting users to a broken provider authorize URL.

**Suggested title:** `fix(backend): reject OAuth authorize when credentials or redirect URI are misconfigured`

**Test plan:**

- Authorize endpoint returns `503` when `{PROVIDER}_CLIENT_ID` is unset.
- Authorize endpoint returns `503` in production when redirect URI contains `localhost`.

**Fork note:** Error messages may mention Railway; generalize to “set env vars and redeploy” before opening upstream PR.

## Opening PRs against upstream

1. Create a branch from fresh `the-momentum/open-wearables` `main` (not this fork’s `main` if it contains unrelated commits).
2. Cherry-pick or replay only the commits for one PR scope above.
3. Push to your GitHub fork and open PR to `the-momentum/open-wearables:main`.
4. After merge upstream, `git fetch origin && git merge origin/main` on this fork.
