The Open Wearables **remote MCP** (`streamable-http`) ships as a **standalone Docker image** built from **`mcp/Dockerfile`**. Deploy it as a **new service** in the same Railway project as your backend (or another project), with its **own** public URL.

<Note>
  **Two services if you need both integrations:** use one Railway service with **`MCP_AUTH_MODE=bearer`** for [Managed Agents](/mcp-server/managed-agents-remote) and a **second** service with **`MCP_AUTH_MODE=oauth`** for the [Claude custom connector](/mcp-server/claude-custom-connector). A single container cannot serve both auth modes.
</Note>

## Prerequisites

- Railway project with a running **Open Wearables backend** (you need a public **`OPEN_WEARABLES_API_URL`** and an **`OPEN_WEARABLES_API_KEY`**).
- GitHub repo connected to Railway (your fork is fine).

## 1. Create the service

<Steps>
  <Step title="Add a service">
    In your Railway project, click **New** â†’ **GitHub Repo** (or **Empty Service** â†’ connect repo), choose the repository that contains **`mcp/Dockerfile`**.
  </Step>

  <Step title="Set root directory">
    Open the new service â†’ **Settings** â†’ **Build** â†’ **Root Directory** â†’ set **`mcp`**.

    Railway should detect **`Dockerfile`** inside that folder and build with:

    - Context: `mcp/`
    - File: `mcp/Dockerfile`
  </Step>

  <Step title="Expose HTTP">
    **Settings** â†’ **Networking** â†’ **Generate Domain** (or attach a custom domain).

    Your MCP endpoint for Anthropic will be:

    `https://<this-service-domain>/mcp`

    unless you change **`MCP_HTTP_PATH`** (default **`/mcp`**).
  </Step>
</Steps>

## 2. Environment variables

In the MCP service â†’ **Variables**, add:

| Variable | Required | Description |
|----------|----------|-------------|
| **`OPEN_WEARABLES_API_URL`** | Yes | Public backend base URL, e.g. `https://backend-production-xxxx.up.railway.app` (no `/mcp`). |
| **`OPEN_WEARABLES_API_KEY`** | Yes | Valid API key; MCP sends **`X-Open-Wearables-API-Key`** to the backend. |
| **`MCP_AUTH_MODE`** | Yes | `oauth` for [Claude custom connector](/mcp-server/claude-custom-connector); `bearer` for [Managed Agents](/mcp-server/managed-agents-remote). |

**When `MCP_AUTH_MODE=oauth` (Claude mobile / claude.ai):**

| Variable | Required | Description |
|----------|----------|-------------|
| **`MCP_PUBLIC_BASE_URL`** | Yes | HTTPS origin of this MCP service, e.g. `https://<this-service>.up.railway.app` |
| **`MCP_GITHUB_CLIENT_ID`** | Yes | GitHub OAuth App client ID |
| **`MCP_GITHUB_CLIENT_SECRET`** | Yes | GitHub OAuth App secret; callback `https://<host>/auth/callback` |

**When `MCP_AUTH_MODE=bearer` (Managed Agents only):**

| Variable | Required | Description |
|----------|----------|-------------|
| **`MCP_BEARER_TOKEN`** | Yes | Same value as Anthropic vault **`static_bearer`** **`token`**. |

Railway injects **`PORT`** automatically â€” the server listens on **`PORT`** (you do not set **`MCP_HTTP_PORT`** unless you know you need it).

Optional:

| Variable | Default | Purpose |
|----------|---------|---------|
| **`MCP_HTTP_PATH`** | `/mcp` | Must match the path in your public MCP URL. |
| **`MCP_HTTP_HOST`** | `0.0.0.0` | Bind address inside the container. |
| **`MCP_JWT_SIGNING_KEY`** | â€” | Stable OAuth JWT signing key across restarts. |
| **`USER_LOCAL_TIMEZONE`** | `America/Los_Angeles` | **`start_local` / `end_local`** when workouts lack **`zone_offset`**. |
| **`LOG_LEVEL`** | `INFO` | Logging. |
| **`REQUEST_TIMEOUT`** | `30` | Backend HTTP timeout (seconds). |

Do **not** commit a production `.env` with these values into git. Keep secrets in Railway **Variables** (or references / secrets stores).

## 3. Health check (recommended)

Configure the service health check to **`GET /health`** on the public URL (path **`/health`** only â€” no Bearer header required).

## 4. Deploy and verify

1. Trigger a deploy; wait for **Active**.
2. Browser or curl:

   ```bash
   curl -sS https://<your-mcp-service>.up.railway.app/health
   ```

   Expect `{"status":"ok"}`.

3. Confirm **`/mcp`** returns **401** without **`Authorization`** and accepts Bearer when set:

   ```bash
   curl -sS -o /dev/null -w "%{http_code}\n" https://<host>/mcp
   curl -sS -o /dev/null -w "%{http_code}\n" \
     -H "Authorization: Bearer $MCP_BEARER_TOKEN" \
     https://<host>/mcp
   ```

## 5. Connect Claude

| Goal | Guide |
